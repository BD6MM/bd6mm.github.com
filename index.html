LNMP_v1.3开启SSL方法

回复 分享

今天网站更换成了香港沙田VPS服务器，重新部署Lnmp环境时SSL访问出错，Google一番资料后发现几段代码可以完美解决。
1、把生成的SSL证书上传至服务器；
2、vi /usr/local/nginx/config/nginx.con ，打开Nginx配置文件；
3、在server{}里加入：

listen 443 ssl;
ssl_certificate /home/wwwroot/ssl/.crt;（上传SSL证书的绝对路径）
ssl_certificate_key /home/wwwroot/ssl/.key;（上传SSL证书的绝对路径）
ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
fastcgi_param HTTPS on;

4、运行lnmp restart命令，重启lnmp即可。

亚马逊AMS使用ROOT+密码登陆方法

回复 分享

1、先用AMS的密钥对登陆;
2、执行

sudo passwd root

为root用户设置密码；
3、执行su输入密码，切换到root用户，然后

sudo passwd ubuntu

为ubuntu用户设置好密码;
4、

sudo chmod 777 /etc/ssh/sshd_config

为sshd_config配置文件设置权限;
5、

vi /etc/ssh/sshd_config

打开sshd_config配置文件;
6、找到PasswordAuthentication，值改为yes;
7、找到PermitRootLogin，值改为yes;
8、

sudo chmod 644 /etc/ssh/sshd_config

9、service sshd restart重启sshd服务或者重启VPS，大功告成！
